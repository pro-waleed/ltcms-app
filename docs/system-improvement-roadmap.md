# خطة تطوير نظام LTCMS

## الخطوة 1: التثبيت السريع

- إصلاح النصوص العربية المعطوبة في الواجهات والرسائل والبذور.
- تصحيح رسائل النجاح والفشل لتطابق العملية المنفذة.
- إيقاف العمليات التي تعدل البيانات عبر `GET`.
- تفعيل المستخدم النشط وتسجيل آخر دخول.

## الخطوة 2: إعادة هيكلة المنطق

- نقل التحقق إلى `FormRequest`.
- نقل منطق الاستيراد والمزامنة إلى `Services`.
- إنشاء `Enums` لحالات الفرص والترشيحات والطلبات.
- توحيد الرسائل والمخرجات.

## الخطوة 3: تحسين الأعمال

- بناء دورة حياة متكاملة للفرصة.
- بناء معايير مفاضلة عادلة للترشيحات.
- تطوير التقارير لتقيس الأثر والعدالة لا مجرد العدد.
- إضافة سجل تدقيق لكل تغيير حالة.

## الخطوة 4: النضج التقني

- كتابة اختبارات Feature وUnit حقيقية.
- إضافة فهارس وتحسين أداء التقارير والبحث.
- فصل الواجهة إلى مكونات قابلة لإعادة الاستخدام.
- توثيق التشغيل والإدارة والمستخدم.

## المخرجات الحالية

- تقرير مراجعة قابل للاستعراض داخل المشروع:
  - `public/system-review.html`
- نسخة نصية مرجعية للخطة:
  - `docs/system-improvement-roadmap.md`
